Memo Outline
------------

Foreword:
Goal is to encourage reproducible research for MEE with config files.

Summary:
Current pipeline works, but has many adjustable parameters.  Setting these
correctly and setting them the same way for multiple analysis runs is a
challenge.

The solution: TOML input files.  These are human readable, with comments.  A set
of config files defines the entire analysis run.  No parameters need be adjusted
in the GUI; they are loaded from the input files.  Also store the exact
configuration used to analyze the data with each run (in case there were
adjustments after loading the config file).

A complete and successful run on the Station 1 sample data was carried out using
only TOML input files.

Discussion:
A full run on Station 1 sample data implemented with the following files:

```
../configs/
├── station1-sample-find-stars-calibration.toml
├── station1-sample-find-stars-eclipse.toml
├── station1-sample-compute-distortion-zenith.toml
├── station1-sample-compute-distortion-eclipse.toml
└── station1-sample-fit-data.toml
```

The first two are entered in Tab 1, the second two in Tab 2, the third in Tab 3.

Kludge: because of time-stamped output files in the current GUI, it is possible
to know ahead of time what the name of the input files for later tabs will be.
The current fix is to use the most recent .zip in the output folder of the
appropriate name (`centroids*.zip` or `distortion*.zip`).  For a single run,
this is guaranteed to work, because there will be no prior `.zip` files, but lax
adherance to good practice could circumvent this fix.

Loading an input file is a two-step process.  A file field with "Browse" and
"Load" buttons was added at the top of each tab of the GUI.  "Browse" allows the
user to identify a config file; "Load" imports all the options from this file
and updates the GUI fields.  It also pops up a window indicating this action has
been taken.

To improve readability of the GUI, a larger font size and scrollbars were added.

Future work - This workflow will allow controlled, reproducible studies of
how variations in the data analysis procedure affect the computed deflection.

- How does choice of calibration field affect distortion coefficients and final results?
- How does the number of eclipse images used affect the final results?
- Which parameters have the most significant impact on the final precision?
