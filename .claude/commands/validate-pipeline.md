# validate-pipeline

Compare CLI pipeline output against golden reference output generated by the original GUI.

## Usage

`/validate-pipeline [run-name]` — compares output in `tests/output/<run-name>/` against `tests/golden/<run-name>/`

If no run-name is given, list available golden runs and prompt the user to choose one.

## Golden output

Golden files are generated by running the original GUI on the test data with a documented config, then copying the output to `tests/golden/<run-name>/`. Each golden directory should contain a `README.md` recording: date generated, GUI version, config used, input data path.

**Ready-to-seed golden reference:** `/Users/jesse/scratch/eddington/output/oas-poster/` contains a complete 5-step Station 1 GUI run (zenith + eclipse, quintic fit). Copy to `tests/golden/station1-oas-poster/` and create a README to make it available for validation.

## Logic

### Step 1 — Check for golden output

Look for `tests/golden/`. If the directory does not exist or is empty:

> No golden reference output found.
>
> To generate it:
> 1. Run the MEE2024 GUI on one of the test datasets (see CLAUDE.md for paths).
> 2. Copy the output files to `tests/golden/<run-name>/`.
> 3. Create `tests/golden/<run-name>/README.md` recording the date, GUI version (`mee2024.__version__` or git commit), config settings used, and input data path.
> 4. Re-run `/validate-pipeline <run-name>`.

Stop here if golden output is missing.

### Step 2 — Check for CLI output

Look for `tests/output/<run-name>/`. If missing, remind the user to run the CLI scripts first and specify `tests/output/<run-name>/` as the output directory.

### Step 3 — Stage-by-stage comparison

For each stage with both golden and CLI output present:

#### find-stars
- **Star count**: number of rows in `*_centroids.csv`. Report CLI vs. golden count; flag if they differ by more than 5%.
- **Centroid positions**: for stars matched between the two catalogs (by nearest-neighbor within 2 pixels), compute RMS positional difference. Flag if > 0.5 px.
- Report: N stars matched, N unmatched in CLI output, N unmatched in golden.

#### compute-distortion (plate solve results)
- **Plate scale** (arcsec/pixel): flag if relative difference > 0.1%.
- **Pointing** (RA, Dec of field center): flag if separation > 1 arcmin.
- **Roll angle**: flag if difference > 0.1°.
- **Distortion coefficients**: report RMS relative difference across all coefficients; flag if > 1%.

#### fit-data
- **Deflection measurement**: report CLI value ± uncertainty vs. golden value ± uncertainty.
- Flag if the values differ by more than 1σ (using the golden uncertainty as the scale).
- Report number of stars used in the fit.

### Step 4 — Summary

Print a pass/fail table. A stage passes if all its checks are within tolerance. Overall pass requires all stages to pass.

## Tolerances (summary)

| Quantity | Tolerance |
|----------|-----------|
| Star count | ±5% |
| Centroid RMS | < 0.5 px |
| Plate scale | < 0.1% relative |
| Pointing | < 1 arcmin |
| Roll angle | < 0.1° |
| Distortion coefficients | < 1% RMS relative |
| Deflection | within 1σ of golden |
